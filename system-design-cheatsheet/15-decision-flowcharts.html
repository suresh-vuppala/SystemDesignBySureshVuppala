Sync vs Async Communication
QuestionSync (HTTP/gRPC)Async (Queue/Event)
Need immediate response?Yes → SyncNo → Async
Can caller wait?Yes → SyncNo → Async (202 + poll)
Tight coupling OK?Yes → SyncNo → Async (decoupled)
Need retry/replay?Manual retryBuilt-in (Kafka offset, SQS DLQ)
Monolith vs Microservices
QuestionMonolithMicroservices
Team size?<10 devs → Monolith>10 devs, multiple teams → Micro
Deploy speed?Single deploy (simple)Independent deploys per service
Complexity budget?Low → MonolithCan handle distributed complexity → Micro
Scaling needs?Scale entire appScale hot services independently
Data consistency?Single DB, ACID easyDistributed txns (Saga), eventual consistency
SQL vs NoSQL (Simplified)
QuestionSQLNoSQL
Need ACID + JOINs?Yes → SQLNo
Schema known upfront?Yes → SQLEvolving → NoSQL (flexible schema)
Write throughput >100K/sec?Hard (single leader)Yes → Cassandra/DynamoDB
Need horizontal scale?Sharding is painfulBuilt-in (masterless, auto-shard)
Complex queries?SQL excels (JOINs, aggregations)Must denormalize, no JOINs
Microservices Patterns: Strangler Fig — gradually replace monolith pieces with services. BFF — separate API gateway per client type. Sidecar — attach proxy (Envoy) to each service for cross-cutting concerns. DDD — bounded contexts define service boundaries (each service owns its domain + data).
Data Privacy: GDPR — EU data stays in EU, right to deletion, consent required. PII handling — encrypt at rest + transit, mask in logs, tokenize for analytics. Data residency — geo-sharding ensures compliance. Audit trail for all access.
