📊 13 · OBSERVABILITY

Logging

Structured JSON logs with correlation IDs. Levels: DEBUG → INFO → WARN → ERROR → FATAL

Guarantees: Immutability (forensics, compliance). Searchability. Correlation IDs trace flow across services. Never log passwords/PII. Tools: ELK, Splunk, Loki, CloudWatch. Sample: 100% errors, 1-10% normal.

Metrics

Golden Signals (Google SRE): Latency · Traffic · Errors · Saturation

SignalWhat It MeasuresKey MetricsAlert When
LatencyTime to serve a requestp50 · p95 · p99 · p999. Never use avg — hides tail latencyp99 > SLA threshold
TrafficDemand on the systemRPS · TPS · bytes/sec · active connectionsSudden spike or unexpected drop
ErrorsRate of failed requestsError rate % · 5xx count · timeout count · DLQ depthError rate > 1% or any 5xx spike
SaturationHow full / overloaded the system isCPU % · Memory % · Disk I/O · Thread pool queue depthCPU > 80% sustained
MethodFull FormBest ForMetrics
REDRate · Errors · DurationMicroservices, APIsRequests/sec · Error % · p99 response time
USEUtilization · Saturation · ErrorsInfrastructure — CPU, memory, disk, networkCPU % · queue depth · device error count
p-values explained: p50 — median. p95 — 95% of requests faster. p99 — SLA boundary. p999 — worst-case outliers. Average hides outliers.
SLI / SLO / SLA: SLI (indicator) — actual measured metric. SLO (objective) — internal target. SLA (agreement) — contractual commitment with penalty. SLO should be stricter than SLA.
Tools: Prometheus — scrape + store time-series. Grafana — dashboards + alerting. Datadog — managed APM. CloudWatch — AWS native. OpenTelemetry — vendor-neutral standard.
One-liner: Track p99 not average — use RED for services, USE for infra, alert on Golden Signals, define SLOs stricter than SLA.

Distributed Tracing

Track single request across services. Trace ID → Spans → Parent-child relationships → Latency breakdown.

Guarantees: Request correlation across all services. Bottleneck identification. Dependency mapping. Tools: OpenTelemetry, Jaeger, Zipkin, Datadog APM. Sample: 100% errors, 1-5% successes.

Monitoring & Alerting

Collect → Alert → Respond → Root Cause Analysis

Alert Rules: Only page if human action needed. Good: "error rate >1% for 2min". Bad: "any single error". Error Budget: 100% - SLO = budget for experimentation/risk. Tools: PagerDuty, Opsgenie, Grafana OnCall.
