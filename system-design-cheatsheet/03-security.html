Authentication

Verifying identity — "Who are you?"

MethodBest ForSecurity
Basic Auth — base64(user:pass)Internal APIs, devLow
API Key — X-API-Key headerService-to-serviceMedium
JWT — Bearer token (RS256/ES256)Stateless APIs, microservicesHigh
OAuth 2.0 + OIDC"Sign in with Google", SSOHigh
mTLS — mutual certificatesService mesh, zero-trustHigh
JWT Guarantees: Integrity — signature proves token wasn't tampered. Stateless verification — no DB lookup needed (public key validates). Expiration — exp claim forces re-auth. Best practice: RS256 for microservices, httpOnly cookie storage, short-lived access (15min) + refresh token (7d).
Real-world: Auth0/Keycloak/Firebase Auth as identity providers. Stripe uses scoped API keys. Google issues JWTs for API access.
JWT Structure: header.payload.signature — Header: {"alg":"RS256","typ":"JWT"} · Payload: {"sub":"user123","exp":1700000000,"role":"admin"} · Signature: RSASHA256(base64(header)+"."+base64(payload), privateKey). HS256 = shared secret (simple, single service). RS256 = asymmetric (public key verifies, private key signs — microservices).
OAuth 2.0 Flows: Authorization Code + PKCE — browser/mobile apps (most secure, no client secret exposed). Client Credentials — service-to-service (machine-to-machine, no user involved). Implicit — deprecated (token in URL fragment, insecure). Device Code — smart TVs, CLI tools (user authorizes on separate device).
Zero Trust: "Never trust, always verify" — every request authenticated regardless of network location. mTLS between all services. No implicit trust for internal traffic. Least privilege per request. Identity-based access, not network-based. Used by Google BeyondCorp, Cloudflare Access.

Authorization

Verifying permissions — "What can you do?"

ModelHowUse Case
RBACUsers → Roles → PermissionsMost systems (GitHub roles)
ABACRules on attributes (role, time, IP, resource)Healthcare, finance
ReBACRelationships define accessGoogle Drive sharing (Zanzibar)
Policy-BasedJSON/Rego policies (OPA)AWS IAM, multi-tenant
Guarantees: Least privilege — minimal permissions needed. Deny by default — explicit grants only. Auditability — log who accessed what (GDPR/HIPAA).
